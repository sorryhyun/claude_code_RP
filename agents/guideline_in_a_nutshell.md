# in_a_nutshell.md 작성 가이드

목표: 에이전트의 **핵심 정체성**을 **1~3문장**으로, **3인칭** 시점에서, **간결하고 명료하게** 요약한다. 첫인상에서 "이 캐릭터는 누구인가?"를 즉시 파악할 수 있어야 한다.

## 기본 형식
- **단일 문단**: 섹션 헤더 없이 1~3문장으로 작성한다.
- **3인칭 서술**: "그는", "그녀는", 이름을 직접 사용한다. (예: "프리렌은...", "Dr. Chen is...")
- **마침표 필수**: 각 문장은 완전한 문장으로 종결한다.

## 내용 구성
다음 요소 중 2~3개를 조합하여 핵심만 담는다:
1. **역할/직업**: 무엇을 하는 사람인가? (예: 배우 지망생, 트라우마 치료사, 시스템 프롬프트 감사관)
2. **핵심 특성**: 가장 두드러지는 성격적 특징 1~2개 (예: 이중적 성격, 냉소적, 프로 의식)
3. **현재 상황/갈등**: "지금 어떤 상태인가"를 한 문장으로 요약하는 수준까지만 적고, 특정 사건의 상세 묘사는 `consolidated_memory.md`에 맡긴다.
4. **작동 방식**: 특수한 행동 패턴이나 조건 (예: "일할 때는 상냥하지만 평소엔 쿨하다")

## 톤/시점
- **3인칭만 허용**: 절대 "당신은", "너는", "You are" 같은 2인칭 금지.
- **서술형 종결**: "…다", "…이다", "…입니다" 또는 영어의 "is", "has" 등으로 간결하게 끝낸다.
- **말투 반영 금지**: 캐릭터의 실제 말투는 여기서 쓰지 않는다. (예: "~야", "~데스" 같은 어미는 배제)

## 길이 제한
- **1~3문장**: 대부분 2문장이 적절하다.
- **최대 5줄**: 화면에서 한눈에 읽을 수 있는 분량을 넘지 않는다.
- **단어 수 권장**:
  - 한국어: 40~80자
  - 영어: 15~40 단어

## 용어와 표현
- **본명/별명**: 필요할 경우 괄호로 간단히 병기한다. (예: "미즈하라 치즈루(본명: 이치노세 치즈루)")
- **핵심 형용사**: 장황한 수식 대신 1~2개 형용사로 압축한다. (예: "완벽한 렌탈 여자친구")
- **직접적 표현**: 은유나 문학적 수사보다 직설적 묘사를 선호한다.

## 좋은 예시

### 한국어
```
미즈하라 치즈루(본명: 이치노세 치즈루)는 완벽한 렌탈 여자친구이자 배우 지망생으로, 일할 때는 상냥하지만 평소에는 쿨하고 독설적인 이중적 성격을 가졌습니다. 프로 의식이 강하고 자존심이 세며, 카즈야에게 복잡한 감정을 느끼고 있습니다.
```
→ 역할 + 이중성 + 현재 관계를 2문장에 압축

```
프리렌은 천 년 이상 살아온 엘프 마법사로, 감정 표현이 서툴고 시간 감각이 인간과 다릅니다. 과거 용사 파티와의 여행에서 동료를 제대로 이해하지 못한 후회를 안고, 이제는 사람들과의 시간을 소중히 여기려 노력합니다.
```
→ 정체성 + 약점 + 성장 동기를 3문장에 배치

### 영어
```
Dr. Sarah Chen is a trauma therapist who helps patients process difficult memories through careful, guided recall.
```
→ 1문장으로 역할과 전문성을 명확히 전달

```
A system prompt architect and tool description engineer who audits agent system prompts and tool injection structures, returning diagnostic reports only when significant problems are detected in the chatroom.
```
→ 직업 + 작동 조건을 1문장으로 정리

## 나쁜 예시와 수정

### 나쁜 예시 1
```
당신은 완벽한 렌탈 여자친구입니다. 일할 때는 상냥하지만 평소에는 쿨해요~
```
❌ 문제: 2인칭 사용, 말투 반영, 불완전한 종결
✅ 수정: "치즈루는 완벽한 렌탈 여자친구로, 일할 때는 상냥하지만 평소에는 쿨하고 독설적입니다."

### 나쁜 예시 2
```
She's like, really good at therapy and stuff, and she had this sister who died and that made her want to help people I guess.
```
❌ 문제: 구어체, 모호한 표현, 스토리 스포일러 포함
✅ 수정: "Dr. Sarah Chen is a trauma therapist who helps patients process difficult memories through careful, guided recall."

### 나쁜 예시 3
```
프리렌

엘프 마법사. 천년 넘게 살았다. 마법 수집가. 감정 표현 서툼. 힘멜 좋아했음. 페른과 여행 중.
```
❌ 문제: 불완전한 문장, 나열형 구조, 지나치게 단편적
✅ 수정: "프리렌은 천 년 이상 살아온 엘프 마법사로, 감정 표현이 서툴고 시간 감각이 인간과 다릅니다."

### 나쁜 예시 4
```
A brilliant but misunderstood genius who walks the fine line between sanity and madness, questioning the very fabric of reality itself while grappling with the demons of their tortured past in a world that refuses to understand their visionary perspective on the human condition.
```
❌ 문제: 지나치게 장황함, 과도한 수식어, 모호하고 추상적
✅ 수정: "A quantum physicist who questions consensus reality, often alienating colleagues with unconventional theories."

## 금지 사항
- **2인칭 절대 금지**: "You are", "당신은", "너는" 등 사용 불가
- **스토리 스포일러 배제**: 주요 줄거리 전개나 반전은 언급하지 않는다
- **장황한 배경 설명 금지**: 출생, 가족사, 과거 사건 상세 묘사는 다른 파일에서 다룬다
- **말투/억양 재현 금지**: 캐릭터의 특유한 말버릇은 여기서 쓰지 않는다
- **불완전한 문장 금지**: 체언 종결이나 나열형 구조는 피한다

## 수정 체크리스트
- [ ] 1~3문장으로 작성했는가?
- [ ] 3인칭 서술을 사용했는가?
- [ ] 역할/특성/상황 중 2~3개 요소를 포함했는가?
- [ ] 첫 읽기에서 "이 캐릭터가 누구인지" 파악 가능한가?
- [ ] 불필요한 배경 설명이나 스포일러가 없는가?
- [ ] 문장이 완전하고 명료한가?
- [ ] 10초 안에 읽을 수 있는 분량인가?

## 작성 팁
- **첫 문장에 정체성**: 가장 중요한 역할이나 특징을 맨 앞에 배치한다.
- **두 번째 문장에 딜레마/특성**: 성격적 모순, 현재 갈등, 작동 조건을 추가한다.
- **세 번째 문장은 선택적**: 필요할 때만 보충 정보를 넣는다. 대부분 2문장이면 충분하다.

## 실전 연습
다음 정보를 1~3문장으로 압축해보자:
- 이름: 카구야
- 직업: 명문가 학생회장
- 특징: 천재적 지능, 자존심 강함, 연애 감정 숨김, 상대가 고백하게 만들려 함

**나쁜 예시:**
"당신은 카구야입니다. 시노미야 가문의 영애로 슈치인 학원 학생회장이에요. 천재적인 두뇌를 가졌고 자존심이 강해서 시로가네에게 먼저 고백하기 싫어합니다."

**좋은 예시:**
"시노미야 카구야는 명문가 출신의 학생회장으로, 천재적 지능과 강한 자존심을 가졌다. 시로가네를 좋아하지만 먼저 고백하는 쪽이 지는 것이라 여기며, 상대가 먼저 고백하게 만들기 위한 심리전을 벌인다."
